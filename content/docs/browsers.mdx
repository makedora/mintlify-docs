---
title: Cloud Browsers
description: Create and control browser instances via CDP
---

Kernel browsers are cloud Chromium instances accessible via the Chrome DevTools Protocol. Each browser runs in its own isolated VM.

## Creating a browser

```typescript
import Kernel from "@onkernel/sdk";

const kernel = new Kernel();
const browser = await kernel.browsers.create();
console.log(browser.cdp_ws_url); // ws://...
```

The `create()` call returns immediately with a `cdp_ws_url` you can pass to Playwright or Puppeteer.

## Connecting with Playwright

```typescript
import { chromium } from "playwright";

const pw = await chromium.connectOverCDP(browser.cdp_ws_url);
const page = pw.contexts()[0].pages()[0];
await page.goto("https://example.com");
```

<Warning>
Always use `connectOverCDP()` — not `connect()`. Kernel browsers expose a CDP endpoint, not a Playwright-native one.
</Warning>

## Session timeouts

By default, sessions have a limited lifetime. For long-running tasks, configure a longer timeout:

```typescript
const browser = await kernel.browsers.create({
  timeout: 3600 // seconds — up to 72 hours (259200)
});
```

## Key details

| Feature | Value |
|---------|-------|
| Protocol | Chrome DevTools Protocol (CDP) |
| Browser | Chromium |
| Isolation | Individual VM per browser |
| Max session | 72 hours |
| Connection | `connectOverCDP(cdp_ws_url)` |
| Observability | Video replays, live view |
| Stealth mode | Built-in anti-detection |
